【Git 関連メモ】

1. リモート（GitHubなど）の最新の変更を、自分が書いたコードの『下』にスッと差し込んで、履歴を一本道にするコマンド
   git pull origin main --rebase

   ※解説：
     リモート（GitHub等）の最新コミットを取り込みつつ、自分のローカルのコミットをその「上に」積み直します。
     これにより、マージコミットを減らし、履歴を一本の直線（リニア）に保てます。

2. リモートの最新版を自分に取り込んで、履歴をきれいに整える。
   git pull --rebase

   ※解説：
     上記と同様ですが、リモートブランチ名（main）を省略した形です。
     現在チェックアウトしているブランチに対して、対応するリモートブランチを rebase で取り込みます。

3. ローカルブランチをGitHubにプッシュするコマンド
   git push origin main

   ※解説：
     ローカルの main ブランチの内容を、リモート（origin）の main ブランチに反映します。
     「なかなかGitHubにファイルがアップロードされないな」というときは、このコマンドで強制的に送信できます。

4. gitのメアドと名前を設定するコマンド（""の中を編集してターミナルに貼り付けて実行）
   git config --global user.email "新しいメールアドレス"
   git config --global user.name "新しいユーザー名"

   ※解説：
     コミット時に記録されるユーザー名とメールアドレスをグローバル設定として登録します。
     --global を付けると、そのPC上のすべてのリポジトリで共通の設定になります。
     プロジェクトごとに異なる設定にしたい場合は、リポジトリ内で --local を使うか、--global を外して設定します。

【Python 仮想環境関連メモ】

1. pythonの仮想環境を構築する方法（VSCode 利用）

   - VSCodeで "Shift + Ctrl + P" を押す
   - 左端が「>」の検索窓が出るので、「python」と入力
   - 「Python: インタープリターを選択」を選ぶ
   - 「仮想環境の作成」を押し、venv を選択
   - Pythonのバージョンを 3.12.10 に合わせる

   ※解説：
     プロジェクトごとに独立したPython環境（venv）を作成することで、
     パッケージのバージョン競合を防ぎ、開発環境をクリーンに保てます。

2. pythonの仮想環境を起動するコマンド（Windows / PowerShell 想定）
   .venv\Scripts\Activate.ps1

   ※解説：
     作成した仮想環境（.venv）を有効化します。
     ターミナルのプロンプトの先頭に (.venv) などと表示されれば、仮想環境がアクティブになっています。

3. パッケージを requirements.txt から一括インストールするコマンド
   pip install -r requirements.txt

   ※解説：
     仮想環境を立ち上げた状態で実行すると、
     requirements.txt に記載されたパッケージとバージョンが一括でインストールされます。
     チーム開発や再現環境構築でよく使います。

4. 逆に、インストールされたパッケージをファイルに残すコマンド
   pip freeze > requirements.txt

   ※解説：
     現在の仮想環境にインストールされているパッケージとバージョンを
     requirements.txt に書き出します。
     環境のスナップショットを取るイメージで、後から同じ環境を再現する際に便利です。
